üìò Chapter 3.3 ‚Äî Dynamic Programming (DP)
üîπ 1. Introduction

DP solves problems by breaking them into overlapping subproblems and storing answers to avoid repetition.

Used when:

‚úî Optimal substructure
‚úî Overlapping subproblems
üîπ 2. DP Concepts
‚≠ê Optimal Substructure

Big problem ‚Üí built from small optimal solutions.

Example: Fibonacci, knapsack.
‚≠ê Overlapping Subproblems

Same subproblems repeat many times. DP stores them in table.

üîπ 3. DP Approaches
(A) Top-Down (Memoization)
‚Ä¢ Recursive
‚Ä¢ Stores subproblem answers
(B) Bottom-Up (Tabulation)
‚Ä¢ Iterative
‚Ä¢ Fills table starting from smallest cases
‚Ä¢ Faster, no recursion overhead
üîπ 4. Steps to Build DP
1. Define subproblems
2. Write recurrence relation
3. Choose top-down / bottom-up
4. Create DP table
5. Compute final answer
