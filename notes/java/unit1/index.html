1. Object-Oriented Programming (OOP) Paradigms
What is OOP?
Object-Oriented Programming is a way of writing programs where we organize code around "objects" rather than just functions. Think of objects as real-world things like a car, a phone, or a person. Each object has properties (like color, size) and behaviors (like start, call, walk).

Why OOP? Before OOP, programs were written in a linear way (top to bottom). As programs got bigger, they became messy and hard to manage. OOP helps us organize code better, making it easier to understand, reuse, and maintain.

Key Features of OOP:
1. Class
A class is like a blueprint or template. For example, think of a "Car" class as a blueprint that describes what a car should have (color, model, engine) and what it can do (start, stop, accelerate).

Example: A "Car" class is just a design. You can create many actual cars from this design.
2. Object
An object is an actual instance created from a class. If "Car" is the blueprint, then "my red Toyota" is an object.

Real-life Example:
Class = "Mobile Phone" (general concept)
Object = "My iPhone 13", "Your Samsung S23" (specific phones)
3. Encapsulation
Encapsulation means wrapping data (variables) and code (methods) together, and hiding the internal details. Think of it like a medicine capsule - you know what it does, but you don't need to know what's inside.

Real-life Example: When you use WhatsApp, you can send messages without knowing how the encryption works. The internal details are hidden from you.
4. Inheritance
Inheritance means creating a new class from an existing class. The new class inherits properties and methods from the parent class.

Real-life Example:
Parent Class = "Vehicle" (has wheels, engine)
Child Class = "Car" (inherits wheels, engine + adds doors, AC)
Another Child = "Bike" (inherits wheels, engine + adds handlebars)
5. Polymorphism
Polymorphism means "many forms". The same method name can behave differently based on the context.

Real-life Example: The word "Play" means different things:
â€¢ Play music â†’ start audio
â€¢ Play video â†’ start video
â€¢ Play game â†’ start game
Same word, different actions based on context.
6. Abstraction
Abstraction means showing only essential information and hiding complex details.

Real-life Example: When you drive a car, you use the steering wheel and pedals. You don't need to understand how the engine works internally. The complex engine details are "abstracted" away from you.

3. JVM, JDK, JRE - Understanding the Java Platform

These three terms are very important but can be confusing. Let's understand them step by step:

JVM (Java Virtual Machine)
JVM is like a virtual computer inside your real computer. It runs Java programs.

How it works: When you write a Java program, it gets converted into bytecode (a special format). JVM reads this bytecode and executes it.

Why it's important: JVM makes Java platform-independent. Windows has a Windows JVM, Mac has a Mac JVM, and Linux has a Linux JVM. They all understand the same bytecode!
JRE (Java Runtime Environment)
JRE is a package that contains JVM + libraries needed to run Java programs.

Think of it like: If you want to watch a movie, you need a video player. JRE is like that player for Java programs.

Contains: JVM + Java libraries (pre-written code for common tasks)
JDK (Java Development Kit)
JDK is a complete package for Java developers. It has everything you need to write, compile, and run Java programs.

Contains: JRE + Development Tools (compiler, debugger, etc.)

Simple rule:
â€¢ If you want to RUN Java programs â†’ Install JRE
â€¢ If you want to WRITE Java programs â†’ Install JDK (it includes JRE)
Analogy to Remember:
â€¢ JVM = Engine of a car (runs the program)
â€¢ JRE = Complete car (engine + fuel + basic features to drive)
â€¢ JDK = Car + Toolkit (everything to build, modify, and drive)
How Java Code Runs:
Step 1: Write code in .java file (e.g., Hello.java)
Step 2: Compile using javac (Java compiler) â†’ creates Hello.class (bytecode)
Step 3: JVM reads Hello.class and executes it
Step 4: Program runs and shows output

7. Control Statements

Control statements help us make decisions and repeat actions in our programs. Think of them as traffic signals that control the flow of your code.

1. if-else Statement (Decision Making)
Purpose: Execute code based on a condition (true/false)
Syntax: // Simple if if (condition) { // code runs if condition is true } // if-else if (condition) { // runs if true } else { // runs if false } // if-else-if ladder if (condition1) { // runs if condition1 is true } else if (condition2) { // runs if condition2 is true } else { // runs if all conditions are false } Example - Grade System: int marks = 85; if (marks >= 90) { System.out.println("Grade A"); } else if (marks >= 75) { System.out.println("Grade B"); // This executes } else if (marks >= 60) { System.out.println("Grade C"); } else { System.out.println("Fail"); }
2. switch Statement (Multiple Choices)
Purpose: Select one option from many choices. Better than multiple if-else when checking the same variable.
Syntax: switch (variable) { case value1: // code break; case value2: // code break; default: // code if no case matches } Example - Day of Week: int day = 3; switch (day) { case 1: System.out.println("Monday"); break; case 2: System.out.println("Tuesday"); break; case 3: System.out.println("Wednesday"); // This executes break; default: System.out.println("Invalid day"); }
Important: Don't forget the break statement! Without it, code will continue to the next case (fall-through).
3. for Loop (Repeat Known Times)
Purpose: Repeat code a specific number of times. Use when you know how many iterations you need.
Syntax: for (initialization; condition; update) { // code to repeat } Example - Print 1 to 5: for (int i = 1; i <= 5; i++) { System.out.println(i); } // Output: 1 2 3 4 5 Example - Array Traversal: int[] marks = {85, 90, 78, 92}; for (int i = 0; i < marks.length; i++) { System.out.println("Mark " + i + ": " + marks[i]); } Enhanced for Loop (for-each): for (int mark : marks) { System.out.println(mark); // Simpler for arrays }
4. while Loop (Repeat Until Condition False)
Purpose: Repeat code as long as condition is true. Use when you don't know how many iterations needed.
Syntax: while (condition) { // code to repeat } Example - Count to 5: int i = 1; while (i <= 5) { System.out.println(i); i++; // Important: update counter! } Example - Password Check: String password = ""; while (!password.equals("1234")) { System.out.println("Enter password:"); password = input.next(); } System.out.println("Access granted!");
5. do-while Loop (Execute At Least Once)
Purpose: Similar to while, but guarantees at least one execution (checks condition after running code).
Syntax: do { // code to repeat } while (condition); Example - Menu System: int choice; do { System.out.println("1. Add"); System.out.println("2. Delete"); System.out.println("3. Exit"); choice = input.nextInt(); // Process choice... } while (choice != 3); // Repeat until Exit
When to Use Which Loop?
for: When you know the number of iterations (e.g., print 1-10, traverse array)
while: When condition-dependent (e.g., until correct password)
do-while: When code must run at least once (e.g., menu display)
6. break and continue
break - Exit Loop Immediately: for (int i = 1; i <= 10; i++) { if (i == 5) { break; // Loop stops at 5 } System.out.println(i); } // Output: 1 2 3 4 continue - Skip Current Iteration: for (int i = 1; i <= 5; i++) { if (i == 3) { continue; // Skip 3 } System.out.println(i); } // Output: 1 2 4 5

7. Inheritance and Types of Inheritance
What is Inheritance?
Inheritance is a mechanism where a new class (child/subclass) acquires properties and methods of an existing class (parent/superclass). It promotes code reusability.

Syntax: class Child extends Parent { }
Real-life Example:
You inherit features from your parents (eye color, height, etc.). Similarly, a child class inherits features from its parent class.
Type 1: Single Inheritance
Definition: One child class inherits from one parent class.
Example - Single Inheritance: class Vehicle { void start() { System.out.println("Vehicle started"); } } class Car extends Vehicle { void drive() { System.out.println("Car driving"); } } Usage: Car c = new Car(); c.start(); // Inherited from Vehicle c.drive(); // Own method
Type 2: Multilevel Inheritance
Definition: A child class inherits from a parent class, which itself inherits from another parent class. Forms a chain.
Example - Multilevel Inheritance: class Vehicle { void start() { System.out.println("Vehicle started"); } } class Car extends Vehicle { void drive() { System.out.println("Car driving"); } } class SportsCar extends Car { void turboBoost() { System.out.println("Turbo activated!"); } } Usage: SportsCar sc = new SportsCar(); sc.start(); // From Vehicle (grandparent) sc.drive(); // From Car (parent) sc.turboBoost(); // Own method
Type 3: Hierarchical Inheritance
Definition: Multiple child classes inherit from the same parent class.
Example - Hierarchical Inheritance: class Animal { void eat() { System.out.println("Animal eats"); } } class Dog extends Animal { void bark() { System.out.println("Dog barks"); } } class Cat extends Animal { void meow() { System.out.println("Cat meows"); } } Usage: Dog d = new Dog(); d.eat(); // Inherited d.bark(); // Own Cat c = new Cat(); c.eat(); // Inherited c.meow(); // Own
Type 4: Multiple Inheritance (NOT in Java)
âš ï¸ Important: Java does NOT support multiple inheritance with classes (one class cannot extend multiple classes). This prevents the "Diamond Problem".

Solution: Use interfaces to achieve multiple inheritance (explained in Interface section).
Type 5: Hybrid Inheritance (NOT in Java)
âš ï¸ Important: Hybrid inheritance (combination of two or more types) is also NOT supported in Java with classes.

Solution: Can be achieved using interfaces.
Remember:
â€¢ Java supports: Single, Multilevel, Hierarchical
â€¢ Java does NOT support: Multiple, Hybrid (with classes)
â€¢ Use extends keyword for inheritance
â€¢ A class can extend only ONE class but implement MULTIPLE interfaces

9. Abstract Classes
What is an Abstract Class?
An abstract class is a class that cannot be instantiated (cannot create objects). It serves as a template for other classes. It can have both abstract methods (without body) and concrete methods (with body).
Real-life Example:
"Shape" is an abstract concept. You can't draw a "shape" directly, but you can draw specific shapes like circle, rectangle, triangle. Shape is the abstract class, specific shapes are concrete classes.
Abstract Class Example: abstract class Shape { String color; // Abstract method (no body) abstract void draw(); // Concrete method (has body) void setColor(String color) { this.color = color; } } class Circle extends Shape { // Must implement abstract method void draw() { System.out.println("Drawing circle"); } } class Rectangle extends Shape { // Must implement abstract method void draw() { System.out.println("Drawing rectangle"); } } Usage: // Shape s = new Shape(); // ERROR! Cannot instantiate Circle c = new Circle(); c.setColor("Red"); c.draw(); // Drawing circle Rectangle r = new Rectangle(); r.draw(); // Drawing rectangle
Rules for Abstract Classes:
1. Use abstract keyword
2. Cannot create objects of abstract class
3. Can have abstract methods (no body) and concrete methods (with body)
4. Child class MUST implement all abstract methods (or be abstract itself)
5. Can have constructors and static methods
6. Can have instance variables
When to use Abstract Classes?
â€¢ When you have common functionality to share
â€¢ When you want to enforce certain methods in child classes
â€¢ When you want partial abstraction (some concrete, some abstract methods)

11. Interface
What is an Interface?
An interface is a completely abstract class that contains only abstract methods (and constants). It's like a contract that a class must follow. It achieves 100% abstraction and supports multiple inheritance.
Real-life Example:
Think of a TV remote interface. Different TVs (Samsung, LG, Sony) implement the same interface (power button, volume, channel buttons), but each works differently internally.
Interface Example: interface Animal { // All methods are public abstract by default void eat(); void sleep(); // All variables are public static final (constants) int LEGS = 4; } class Dog implements Animal { // Must implement all interface methods public void eat() { System.out.println("Dog eats"); } public void sleep() { System.out.println("Dog sleeps"); } } Usage: Dog d = new Dog(); d.eat(); System.out.println("Legs: " + Animal.LEGS);
Multiple Inheritance with Interfaces
Multiple Interface Implementation: interface Flyable { void fly(); } interface Swimmable { void swim(); } class Duck implements Flyable, Swimmable { public void fly() { System.out.println("Duck flies"); } public void swim() { System.out.println("Duck swims"); } } Usage: Duck d = new Duck(); d.fly(); d.swim();
Interface Inheritance
Interface can extend another Interface: interface A { void methodA(); } interface B extends A { void methodB(); } class C implements B { public void methodA() { System.out.println("Method A"); } public void methodB() { System.out.println("Method B"); } }
Rules for Interfaces:
1. Use interface keyword
2. All methods are public abstract (by default)
3. All variables are public static final (constants)
4. Cannot have constructors
5. A class can implement multiple interfaces
6. Use implements keyword
7. Must implement ALL methods from interface
Abstract Class vs Interface:

Abstract Class:
â€¢ Can have abstract and concrete methods
â€¢ Can have instance variables
â€¢ Can have constructors
â€¢ Single inheritance only
â€¢ Use extends

Interface:
â€¢ Only abstract methods (before Java 8)
â€¢ Only constants (static final)
â€¢ No constructors
â€¢ Multiple inheritance
â€¢ Use implements
