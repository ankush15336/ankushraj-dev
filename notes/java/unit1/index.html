1. Object-Oriented Programming (OOP) Paradigms
What is OOP?
Object-Oriented Programming is a way of writing programs where we organize code around "objects" rather than just functions. Think of objects as real-world things like a car, a phone, or a person. Each object has properties (like color, size) and behaviors (like start, call, walk).

Why OOP? Before OOP, programs were written in a linear way (top to bottom). As programs got bigger, they became messy and hard to manage. OOP helps us organize code better, making it easier to understand, reuse, and maintain.

Key Features of OOP:
1. Class
A class is like a blueprint or template. For example, think of a "Car" class as a blueprint that describes what a car should have (color, model, engine) and what it can do (start, stop, accelerate).

Example: A "Car" class is just a design. You can create many actual cars from this design.
2. Object
An object is an actual instance created from a class. If "Car" is the blueprint, then "my red Toyota" is an object.

Real-life Example:
Class = "Mobile Phone" (general concept)
Object = "My iPhone 13", "Your Samsung S23" (specific phones)
3. Encapsulation
Encapsulation means wrapping data (variables) and code (methods) together, and hiding the internal details. Think of it like a medicine capsule - you know what it does, but you don't need to know what's inside.

Real-life Example: When you use WhatsApp, you can send messages without knowing how the encryption works. The internal details are hidden from you.
4. Inheritance
Inheritance means creating a new class from an existing class. The new class inherits properties and methods from the parent class.

Real-life Example:
Parent Class = "Vehicle" (has wheels, engine)
Child Class = "Car" (inherits wheels, engine + adds doors, AC)
Another Child = "Bike" (inherits wheels, engine + adds handlebars)
5. Polymorphism
Polymorphism means "many forms". The same method name can behave differently based on the context.

Real-life Example: The word "Play" means different things:
‚Ä¢ Play music ‚Üí start audio
‚Ä¢ Play video ‚Üí start video
‚Ä¢ Play game ‚Üí start game
Same word, different actions based on context.
6. Abstraction
Abstraction means showing only essential information and hiding complex details.

Real-life Example: When you drive a car, you use the steering wheel and pedals. You don't need to understand how the engine works internally. The complex engine details are "abstracted" away from you.
2. Overview of Java
What is Java?
Java is a programming language created by Sun Microsystems (now owned by Oracle) in 1995. It was designed to be simple, secure, and able to run on any device.

Famous Java Slogan: "Write Once, Run Anywhere" (WORA)

This means if you write a Java program on Windows, it will run on Mac, Linux, Android, or any other platform without changes.

Key Features of Java:
‚úì Simple: Easy to learn and write
‚úì Object-Oriented: Based on objects and classes
‚úì Platform Independent: Runs on any operating system
‚úì Secure: Has built-in security features
‚úì Robust: Strong error handling, manages memory automatically
‚úì Multithreaded: Can do multiple tasks at the same time
Note: Java is used to build Android apps, web applications, enterprise software, games, and much more. Popular apps like Minecraft, Netflix (backend), and most Android apps are written in Java!
3. JVM, JDK, JRE - Understanding the Java Platform

These three terms are very important but can be confusing. Let's understand them step by step:

JVM (Java Virtual Machine)
JVM is like a virtual computer inside your real computer. It runs Java programs.

How it works: When you write a Java program, it gets converted into bytecode (a special format). JVM reads this bytecode and executes it.

Why it's important: JVM makes Java platform-independent. Windows has a Windows JVM, Mac has a Mac JVM, and Linux has a Linux JVM. They all understand the same bytecode!
JRE (Java Runtime Environment)
JRE is a package that contains JVM + libraries needed to run Java programs.

Think of it like: If you want to watch a movie, you need a video player. JRE is like that player for Java programs.

Contains: JVM + Java libraries (pre-written code for common tasks)
JDK (Java Development Kit)
JDK is a complete package for Java developers. It has everything you need to write, compile, and run Java programs.

Contains: JRE + Development Tools (compiler, debugger, etc.)

Simple rule:
‚Ä¢ If you want to RUN Java programs ‚Üí Install JRE
‚Ä¢ If you want to WRITE Java programs ‚Üí Install JDK (it includes JRE)
Analogy to Remember:
‚Ä¢ JVM = Engine of a car (runs the program)
‚Ä¢ JRE = Complete car (engine + fuel + basic features to drive)
‚Ä¢ JDK = Car + Toolkit (everything to build, modify, and drive)
How Java Code Runs:
Step 1: Write code in .java file (e.g., Hello.java)
Step 2: Compile using javac (Java compiler) ‚Üí creates Hello.class (bytecode)
Step 3: JVM reads Hello.class and executes it
Step 4: Program runs and shows output
4. Data Types in Java

Data types tell the computer what kind of data we're storing. Think of them as different types of containers.

Two Main Categories:
1. Primitive Data Types (Basic Types)
These are the most basic types, built into Java. They store simple values.

Think of them as: Simple boxes that hold one type of item
Primitive Types Table: Type Size Range Example ---- ---- ----- ------- byte 1 byte -128 to 127 byte age = 25; short 2 bytes -32,768 to 32,767 short year = 2024; int 4 bytes -2 billion to int salary = 50000; 2 billion long 8 bytes Very large numbers long distance = 123456789L; float 4 bytes Decimal numbers float price = 99.99f; double 8 bytes Large decimals double pi = 3.14159; char 2 bytes Single character char grade = 'A'; boolean 1 bit true or false boolean isStudent = true;
Remember:
‚Ä¢ Use int for whole numbers (most common)
‚Ä¢ Use double for decimal numbers (more precise than float)
‚Ä¢ Use boolean for yes/no or true/false conditions
‚Ä¢ Use char for single letters (must use single quotes 'A')
2. Reference Data Types (Objects)
These store references (addresses) to objects. They're more complex than primitives.

Examples: String, Arrays, Classes, Interfaces
Reference Type Examples: String name = "Ankush"; // Text int[] numbers = {1, 2, 3, 4}; // Array Student s = new Student(); // Object
Difference Explained:
Primitive: Stores the actual value directly
Example: int age = 20; (stores 20 directly)

Reference: Stores the memory address where the object is located
Example: String name = "Raj"; (stores address of "Raj" in memory)
5. Variables and Arrays
What is a Variable?
A variable is a named container that stores data. Think of it as a box with a label on it.
Variable Declaration: dataType variableName = value; Examples: int age = 21; // Integer variable String name = "Raj"; // String variable double cgpa = 8.5; // Double variable boolean isPassed = true; // Boolean variable
Rules for Variable Names:
‚úì Must start with a letter, $ or _
‚úì Can contain letters, numbers, $ and _
‚úì Cannot use Java keywords (int, class, if, etc.)
‚úì Case sensitive (age and Age are different)
‚úì Use meaningful names (studentName is better than x)
What is an Array?
An array is a collection of similar items stored together. Think of it as a row of boxes, all holding the same type of item.
Array Declaration: // Method 1: Declare and initialize later int[] marks = new int[5]; // Creates array for 5 integers marks[0] = 85; // First element marks[1] = 90; // Second element // Method 2: Declare and initialize together int[] marks = {85, 90, 78, 92, 88}; // Method 3: Multi-dimensional array int[][] matrix = { {1, 2, 3}, {4, 5, 6} };
Important:
‚Ä¢ Array indices start from 0 (first element is at index 0)
‚Ä¢ Array size is fixed once created
‚Ä¢ Access elements using arrayName[index]
‚Ä¢ Get array length using arrayName.length
Real-life Example:
Think of an array like a row of lockers:
Locker 0: Student 1's books
Locker 1: Student 2's books
Locker 2: Student 3's books
Each locker (index) holds one student's books (data)
