üìò Chapter 1.1 ‚Äî What is an Operating System?
1. Definition of Operating System
Definition: An Operating System (OS) is system software that acts as an intermediary between computer hardware and the user. It manages hardware resources and provides services for application programs.

The operating system is the most important program that runs on a computer. Every general-purpose computer must have an operating system to run other programs and applications.

Key Functions:
Process Management ‚Äì Creating, scheduling, and terminating processes
Memory Management ‚Äì Allocating and deallocating memory space
File System Management ‚Äì Managing files and directories
Device Management ‚Äì Managing input/output devices
Security & Protection ‚Äì Protecting data and resources
2. Types of Operating Systems
1. Batch Operating System

Jobs are processed in batches without user interaction. Similar jobs are grouped together and executed sequentially.

Advantages:
‚Ä¢ No user interaction required
‚Ä¢ Efficient use of CPU time
‚Ä¢ Suitable for large repetitive tasks
Disadvantages:
‚Ä¢ No user interaction possible
‚Ä¢ Difficult to debug
‚Ä¢ Long waiting time for job execution
2. Time-Sharing Operating System

Multiple users can use the system simultaneously. CPU time is divided into small time slices and assigned to each user in round-robin fashion.

Examples: UNIX, Linux, Windows Server
3. Real-Time Operating System (RTOS)

Processes data as it comes in, typically without delays. Used in systems where timing is critical.

Examples: Medical equipment, Industrial control systems, Military applications
4. Distributed Operating System

Uses multiple central processors to serve multiple real-time applications and users. Data processing is distributed across multiple machines.

5. Network Operating System

Runs on a server and provides the capability to manage data, users, groups, security, applications, and other networking functions.

3. Process Management
Process: A program in execution. It includes the program code, current activity, program counter, and stack.
Process States:
1. New: Process is being created
2. Ready: Process is waiting to be assigned to a processor
3. Running: Instructions are being executed
4. Waiting: Process is waiting for some event to occur
5. Terminated: Process has finished execution
Process Scheduling:

The OS must select processes from the job pool and load them into memory for execution.

FCFS (First Come First Serve)
SJF (Shortest Job First)
Priority Scheduling
Round Robin
Multilevel Queue Scheduling
üìò Chapter 1.2 ‚Äî Memory Management
1. Memory Management Basics
Definition: Memory management is the process of controlling and coordinating computer memory, assigning portions called blocks to various running programs to optimize overall system performance.

Main memory (RAM) is a large array of bytes, each with its own address. The CPU fetches instructions from memory based on the program counter value.

Memory Management Techniques:
Contiguous Allocation
Paging
Segmentation
Virtual Memory
2. Paging

Physical memory is divided into fixed-sized blocks called frames. Logical memory is divided into blocks of the same size called pages.

Advantages of Paging:
‚Ä¢ No external fragmentation
‚Ä¢ Memory management becomes easier
‚Ä¢ Swapping is easier
Page Table Entry:
[ Page Number | Frame Number ]

Logical Address:
[ Page Number | Page Offset ]
3. Virtual Memory
Definition: Virtual memory is a memory management technique that provides an "idealized abstraction of the storage resources" that allows programs to be larger than physical memory.

It uses both hardware and software to allow a computer to compensate for physical memory shortages, temporarily transferring data from RAM to disk storage.

Benefits:
‚Ä¢ Programs can be larger than physical memory
‚Ä¢ More programs can run concurrently
‚Ä¢ Less I/O needed for swapping
‚Ä¢ Better CPU utilization
Page Replacement Algorithms:
1. FIFO (First In First Out)
2. LRU (Least Recently Used)
3. Optimal Page Replacement
4. LFU (Least Frequently Used)
üìò Chapter 1.3 ‚Äî File System Management
1. File System Basics
File: A named collection of related information recorded on secondary storage. Files represent programs and data.

A file system provides a way to organize, store, retrieve, and manage information on storage devices. It controls how data is stored and retrieved.

File Operations:
Create ‚Äì Creating a new file
Open ‚Äì Opening an existing file
Read ‚Äì Reading data from a file
Write ‚Äì Writing data to a file
Close ‚Äì Closing a file
Delete ‚Äì Removing a file
2. Directory Structure

A directory is a container that stores files and other directories. It provides a hierarchical organization of files.

Types of Directory Structures:
1. Single-Level Directory: All files in one directory
2. Two-Level Directory: Separate directory for each user
3. Tree-Structured Directory: Hierarchical structure (most common)
4. Acyclic Graph Directory: Allows sharing of subdirectories
3. File Allocation Methods
1. Contiguous Allocation

Each file occupies a set of contiguous blocks on disk. Simple and supports both sequential and direct access.

‚úì Fast access
‚úì Simple implementation
‚úó External fragmentation
‚úó File size must be known in advance
2. Linked Allocation

Each file is a linked list of disk blocks. Blocks may be scattered anywhere on the disk.

‚úì No external fragmentation
‚úì File can grow dynamically
‚úó Only sequential access
‚úó Pointer overhead
3. Indexed Allocation

Brings all pointers together into one location: the index block. Each file has its own index block.

‚úì Supports direct access
‚úì No external fragmentation
